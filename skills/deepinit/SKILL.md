<Deepinit>

## Purpose

Crawl an entire codebase and generate AGENTS.md files at every directory level.
These files help all sub-agents quickly understand what each module does,
reducing onboarding cost for every subsequent task.

Run once per new project. Update when project structure changes significantly.

## Use_When

- First time working on an unfamiliar codebase
- User says "deepinit", "초기화", "map the codebase"
- Atlas/Athena's explore agent reports low familiarity with project structure

## Steps

### Phase 1 — SCAN

```
Task(subagent_type="agent-olympus:explore", model="haiku",
  prompt="Scan the entire project structure:
  1. List all directories with file counts
  2. Identify: src/, test/, config/, docs/ patterns
  3. Note: package.json, Cargo.toml, go.mod, pyproject.toml locations
  4. Report tech stack per directory")
```

### Phase 2 — GENERATE

For each significant directory (has 2+ code files), generate AGENTS.md:

```
Task(subagent_type="agent-olympus:writer", model="haiku",
  prompt="Create AGENTS.md for directory: <dir_path>

  Format:
  # <directory-name>
  <!-- parent: <relative-path-to-parent-AGENTS.md> -->

  ## Purpose
  <1-2 sentences: what this module/directory does>

  ## Key Files
  - <file>: <one-line description>

  ## Dependencies
  - Imports from: <list>
  - Imported by: <list>

  ## Conventions
  - <naming patterns, test patterns, etc.>

  Files in directory: <file_list>")
```

### Phase 3 — ROOT SUMMARY

Generate root AGENTS.md as a **table of contents** (≤100 lines):

```
# <project-name>
<!-- generated by deepinit — treat as TOC, not encyclopedia -->

## Architecture
<3-5 sentence high-level description>

## Dependency Layers
<e.g. Types → Config → Repo → Service → Runtime → UI>

## Directory Map
- src/     — <purpose> → see src/AGENTS.md
- tests/   — <purpose>
- docs/    — knowledge base (architecture, principles, plans)
- config/  — <purpose>

## Tech Stack
<languages, frameworks, build tools>

## Key Commands
- Build: <command>
- Test:  <command>
- Lint:  <command>
- Run:   <command>

## Knowledge Base
- Architecture:      docs/ARCHITECTURE.md (if exists)
- Golden Principles: docs/golden-principles.md (if exists)
- Exec Plans:        docs/exec-plans/ (if exists)
```

**Rule**: AGENTS.md is a map. Deep content goes in docs/. Never exceed 100 lines.
If `docs/golden-principles.md` and `docs/ARCHITECTURE.md` do NOT exist, suggest:
`"Run /harness-init to set up full harness engineering structure."`

### Phase 3.5 — DOCS SCAFFOLD (if docs/ missing, and harness-init not yet run)

If no `docs/ARCHITECTURE.md` exists (harness-init not yet run), create minimal stubs.
**Skip this phase entirely if `docs/golden-principles.md` exists** — harness-init already created richer docs.

```bash
mkdir -p docs/exec-plans
```

```
Task(subagent_type="agent-olympus:writer", model="haiku",
  prompt="Create minimal docs stubs for this project.

  Project: <project_name>, Stack: <tech_stack>

  Create:
  1. docs/ARCHITECTURE.md — 1-page architecture overview with dependency layers
  2. docs/exec-plans/README.md — plans index with Active/Completed/Tech Debt sections

  Keep each file under 50 lines. These are stubs — real content added by harness-init.")
```

### Phase 4 — VERIFY

```bash
find . -name "AGENTS.md" | head -20  # confirm files created
```

## Update Mode

When re-running on an existing project:
- Preserve `<!-- MANUAL -->` sections (user-written content)
- Update auto-generated sections with current file lists
- Add new directories, remove deleted ones

## Integration

Atlas/Athena invoke deepinit in Phase 0 (Auto Project Onboarding)
when AGENTS.md is absent. deepinit runs once per project.

</Deepinit>
