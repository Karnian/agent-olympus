# Agent Audit P0 Follow-up Work Order

Branch: `codex/agent-audit-p0`

Baseline commit: `881f932`

This work order supersedes the external review text used to audit the baseline commit. It
keeps the review's eight findings, but corrects recovery, CLI parsing, routing, and legacy
compatibility requirements that were incomplete or unsafe as originally written.

## Invariants

- Keep concurrency admission fail-closed. Content recovery must not silently authorize work
  whose active reservation state cannot be established.
- Keep read-only worker isolation, review digest binding, and exact AO_REVIEW_V1 parsing.
- Use Node.js built-ins only. Preserve private state modes and atomic state writes.
- Add regression tests for every changed behavior.

## P0-1 — Recoverable, fail-closed concurrency ledger errors

Malformed JSON or malformed fields/entries in a supported schema must be quarantined under the ledger lock. Quarantine
must succeed before the live ledger is replaced, use a collision-safe name, and apply only to a
validated private regular ledger artifact. Unsafe paths, symlinks, hardlinks, wrong ownership or
mode, oversize files, and I/O or permission failures remain blocking errors.

The recovery-aware ledger is schema v2. A valid ordinary v1 ledger is migrated durably to v2;
the v2 boundary makes older v1 readers reject recovery state rather than silently discard its
barrier. Unknown/future versions and v1 artifacts carrying unpublished recovery metadata remain
untouched and blocking instead of being quarantined or interpreted as empty.

After quarantine, reconstruct durable team reservations where possible. Any reservation state
that cannot be established must remain conservatively blocking for the normal stale window; do
not treat the ledger as immediately empty. Re-scan and merge exact durable team identities at the
recovery-barrier boundary so team state written or repaired during that window is accounted for
before admission reopens. Legacy defaults apply only when their fields are absent: explicit null
schema/kind fields are corrupt, as are string, boolean, fractional, or unsafe-integer
`schemaVersion` values. Only unknown safe-integer versions remain untouched and blocking, and any
v1 `recovery` field (including null) remains untouched and blocking. Errors must report stable
state/ledger paths and a concrete remediation. Update hook
documentation to describe the concurrency gate exception to the general fail-safe hook convention.

Quarantine copies must preserve the validated original bytes, including malformed UTF-8. Every
canonical ledger serialization is compact and checked against the 1 MiB read bound before atomic
replacement so migration or reconstruction cannot create a ledger the next reader rejects.
Concurrency `.ao/` and state directories must be owner-private mode `0700` on POSIX.

Tests must cover repeated malformed input, exact-byte quarantine, strict null handling, late durable
team recovery, bounded serialization, owner-private directories, unsafe/unwritable state diagnostics,
collision-safe quarantine names, and the existing fail-closed filesystem checks. SessionEnd must
preserve the live ledger plus lock/reclaim/claim/stale generations regardless of mtime. Its generic
24-hour cleanup applies only to collision-safe `ao-concurrency.json.corrupt-*` quarantine files and
is not the primary recovery path.

## P0-2 — Terminate Claude's variadic allowed-tools option

In both direct Claude argv and tmux command composition, pass one comma-joined allowed-tools
value and an explicit end-of-options marker before the prompt:

```text
--allowedTools Read,Glob,Grep -- PROMPT
```

Tests must prove that there is exactly one allowed-tools value, `--` immediately precedes the
final prompt, and no optional flag is required to keep the prompt from being consumed.

## P1-3 — Provider-aware cross-review routing

Explicit Codex or Gemini actors route to `external-model`, including mixed Claude plus external
provider requests. A generic cross-review with no provider keeps the existing external-model
default. An explicit Claude-only actor must be removed from external-model scoring as well as
the hard override and must route to an internal read-only reviewer, never `/ask`, `unknown`, or
a mutation executor. Cover English and Korean actor forms, including `교차 검증`.

Actor detection must distinguish a requested reviewer from a provider mentioned only as the
provenance or subject of an artifact or earlier review. Reviewing provider-authored output stays
on the internal read-only reviewer, while applying or fixing findings from an earlier Claude,
Codex, or Gemini review stays on a mutation-capable route. Include negative English and Korean
provenance cases such as `review the patch generated by Codex`, `fix issues from the Claude
review`, `We use Codex for code generation; review this adapter`, and
`코덱스가 검토한 문제를 수정해줘`. Imperative forms such as `use Codex to review...` and
`use Codex, review...` remain external-provider assignments. When a prompt both assigns a
provider review and explicitly asks to implement or fix the result, mutation intent wins,
including comma- or semicolon-separated commands. Passive assignments
such as `have this patch reviewed by Codex` remain explicit external-provider requests. Mutation
verbs must be command-shaped; nouns in `review this fix` or `review this update` do not turn the
request into an implementation route.

Tests must assert classification, the final recommended agent, and matching UserPromptSubmit
advice so the initial intent hook cannot contradict the downstream model router.

## P1-4 — AO_REVIEW_V1 file/line coupling

All five routed reviewer documents must state: `line` must be `null` whenever `file` is `null`.
Each embedded JSON example must pass `parseReviewResult()` after substituting a valid digest.
Tests must also assert the sentence is present and explicitly reject `{ file: null, line: 42 }`.

## P2-5 — Codex read-only tmux version gate

Share the authoritative minimum-version check used by the direct Codex adapter. Read-only tmux
composition must fail closed for unknown or older Codex versions before creating its temporary
prompt file, with the same actionable upgrade message. Hermetic tests cover old, unknown, and
supported versions. Both read-only and mutable Codex tmux commands must also terminate `exec`
options with `--` immediately before the final prompt, so option-shaped prompts such as `--help`
cannot be consumed by the Codex CLI parser.

## P2-6 — Legacy tmux session compatibility

Team list/kill operations must recognize both current hashed names and the actual legacy form:

```text
ao-team-<sanitized-team>-<sanitized-worker>
```

Match both current and legacy sessions by exact names derived from the team's durable worker
names. A raw prefix is unsafe across naming generations: the current prefix for `alpha` can
also prefix a legacy session for a crafted team such as `alpha-<hash(alpha)>`. Without worker
identities, team-scoped list/kill must claim no sessions; production shutdown callers provide
the durable names. Test legacy and current sessions, `alpha` versus `alpha-worker`, the crafted
hash-shaped legacy collision, and the no-worker-identities case. Unscoped listing retains its
delimiter-based behavior. Document the remaining unavoidable ambiguity when two original
team/worker identity pairs sanitize to exactly the same legacy name.

## P2-7 — Avoid no-op durable ledger rewrites

Compare canonical state before pruning/operation with canonical state afterward. Skip the ledger
write only when nothing changed. Successful reservations and releases, stale pruning, and legacy
normalization migrations remain durable. Define and test missing-ledger read behavior.

## P2-8 — One AO_REVIEW_V1 contract source

Export frozen contract constants from `review-contract.mjs`. Derive parser sets and
`review-router.mjs` defaults from them. Assert config metadata matches verdicts, result fields,
finding fields, and escalation fields without retyping expected arrays in tests.
