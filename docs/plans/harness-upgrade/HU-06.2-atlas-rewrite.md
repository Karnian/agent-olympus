## The runner wiring (replaces the prose control-flow)

### Top of run ‚Äî replace the `## Loop_Guard` import block (lines 44-84)
The whole `## Loop_Guard` section's direct-call examples are deleted. Replace the
imports with the runner; add the init + resume:
```javascript
import {
  initPipeline, enterPhase, beginAttempt, reattempt, loopTick,
  recordPhaseError, completePhase, skipPhase, reopenPhase,
} from './scripts/lib/phase-runner.mjs';
// runId = active run id (createRun('atlas') / getActiveRunId('atlas')).
const { resumePhase, resumePolicy } = initPipeline(runId, 'atlas');   // AO-CONTRACT:runner-init
// On resume, jump to resumePhase; every earlier enterPhase returns { skip:true }.
```
Keep a short prose note that the runner is the SOLE loop-guard caller and the 15-iter /
3-review / same-error-3√ó bounds are now enforced at the phase boundary. `loop-guard` may
be *mentioned* in prose but MUST NOT be imported or called directly anywhere in the skill.

### Per phase ‚Äî the uniform shape
Every `### Phase X` section is wrapped:
```
const g = enterPhase(runId, '<id>');
if (!g.skip) {
  <existing phase prose, verbatim>
  await completePhase(runId, '<id>');   // checkpoints internally via checkpointIndex
}
```
This DELETES the 7 numeric `saveCheckpoint('atlas',{phase:N,‚Ä¶})` calls (242/423/632/675/
823/837/871) ‚Äî `completePhase` performs the checkpoint write (ledger-first) itself. The
rich payload (`prdSnapshot`/`completedStories`) rides in `completePhase`'s `opts.checkpointData`
where a section needs it (e.g. after `execute`).

### The outer NEVER-STOP loop ‚Äî wraps execute‚Üíverify‚Üíreview
```
beginAttempt(runId);                          // AO-CONTRACT:outer-attempt ‚Äî first attempt tick (15-cap)
//   STOP+escalate "15 iteration limit exceeded" if !allowed
loop {
  enterPhase/‚Äãwork/‚ÄãcompletePhase  execute      // skipped if already complete (review-reject path)
  enterPhase/‚Äãwork/‚ÄãcompletePhase  verify        // internal fix loop uses recordPhaseError
  enterPhase  review
    per round: loopTick(runId,'review') ‚Üí !allowed ‚áí stop review loop, escalate
    ALL APPROVED  ‚Üí completePhase(review); break
    ANY REJECT    ‚Üí reattempt(runId,{reopen:['verify'],reason:'review_reject'})   // AO-CONTRACT:review-reject-reattempt
                    !allowed ‚áí STOP; else continue loop (back to verify)
}
```

## New behavior ‚Äî trivial-path synthetic PRD (rev-3/4 B3)
In `triage`, when COMPLEXITY=trivial: `skipPhase` context/spec/plan and synthesize

and records the US-001 verification ‚Äî NOT a pre-execution self-verify. This keeps the
changelog (passes:true only) + PR body + `checkVerificationGate` all including US-001.
